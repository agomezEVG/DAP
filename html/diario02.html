UNIDAD 2 - SERVICIOS WEB

PROTOCOLO HTTP/S

¿Qué es el protocolo HTTP y HTTP/S?

HTTP (Protocolo de Transferencia de Hipertexto) es un protocolo utilizado para la comunicación entre un cliente y un servidor, cuyo objetivo principal es facilitar la transferencia de información.

Funcionamiento del protocolo

Cliente Web

Solicita e interpreta información

Muestra los resultados en pantalla

Permite la interacción del usuario

Servidor Web

Proporciona contenido estático

Genera contenido dinámico ejecutando programas (por ejemplo, PHP)

Gestiona certificados SSL/TLS

Tipo de solicitudes

GET

Utilizado para solicitar información al servidor sin alterar sus datos.

POST

Empleado para enviar datos al servidor y realizar modificaciones en su contenido.

HTTP / HTTPS

HTTP

Es un protocolo de transferencia sin cifrado, lo que representa un problema de seguridad, ya que los datos transmitidos están expuestos.

HTTPS

Es un protocolo de transferencia con cifrado que soluciona las vulnerabilidades de seguridad de HTTP. Para utilizar HTTPS, se requiere un certificado SSL/TLS.

Códigos de estado

  • 1xx - Mensaje informativo
  • 2xx - Exito
    • 200 - OK
    • 201 - Created
    • 202 - Accepted
  • 3xx - Redirecciones
    • 300 - Multiple Choice
    • 301 - Moved Permanently
    • 302 - Found
  • 4xx - Errores del cliente
    • 400 - Bad Request
    • 401 - Unauthorized
    • 403 - Forbidden
    • 404 - Not Found
  • 5xx - Errores del servidor
    • 500 - Internal Server Error
    • 501 - Not Implemented
    • 502 - Bad Gateway

SERVICIO DNS

¿Qué es el servicio DNS?

El DNS (Domain Name System) es un sistema que traduce nombres de dominio en direcciones IP, permitiendo que los usuarios accedan a servicios de red mediante nombres más fáciles de recordar.

Funcionamiento DNS

Generalmente, en nuestros clientes contamos con una dirección IP que corresponde al servidor DNS, la cual suele ser proporcionada por nuestro proveedor de servicios de Internet (ISP), aunque también es posible configurarla manualmente.

Cuando un cliente desea acceder a un recurso web, como por ejemplo www.google.com, se llevan a cabo los siguientes pasos en los servidores DNS:

  1. Los servidores DNS que reciben la solicitud primero revisan su memoria caché para verificar si ya disponen de la respuesta.
  2. Si la respuesta está almacenada en la caché, la envían de inmediato.
  3. Si no está en la caché, comienzan una búsqueda en otros servidores DNS.
  4. Una vez que encuentran la respuesta, el servidor DNS la almacena en su memoria caché para futuras consultas y la entrega al solicitante.

Tipos de registros

Los registros DNS son responsables de traducir nombres de dominio a direcciones IP. Existen varios tipos de registros, cada uno con funciones específicas:

  • A - Permite traducir nombres a direcciones IPv4
  • AAAA - Permite traducir nombres a direcciones IPv6
  • CNAME - Permite traducir nombres a otros nombres de dominio
  • MX - Permite traducir nombres a servidores de correo
  • NS - Permite traducir nombres a servidores de nombres
  • PTR - Permite traducir direcciones a nombres de dominio
  • SOA - Permite traducir nombres a servidores de autoridad

Resolucion directa e inversa

Resolución directa: Traduce nombres de dominio a direcciones IP.

Resolución inversa: Traduce direcciones IP a nombres de dominio.

Servidor WEB Apache

Características de Apache

  • Servidor web HTTP de código abierto
  • Implementa el protocolo HTTP/1.1
  • Modular
  • Multi-plataforma (Windows, Linux, MacOS)
  • Muy estable

Contextos del Servidor

Apache dispone de contextos de configuración que se heredan y se aplican en diferentes niveles:

Contexto de Servidor

En este contexto se definen las directivas que afectan a todo el servidor.

Contexto de Virtual Host

En este contexto se definen las directivas que afectan a un host virtual (aplicación web).

Contexto de Directorio

En este contexto se definen las directivas que afectan a un directorio específico.

Buenas prácticas en las directivas

Para mantener una buena organización en la configuración de Apache, es recomendable seguir las siguientes buenas prácticas:

  1. DirectoryIndex: Define el archivo que se debe servir cuando se solicita un directorio.
  2. Options: Gestiona las características disponibles en un directorio.
  3. AllowOverride: Permite la configuración de directivas a través de archivos .htaccess.
  4. Order / Require: Regula el acceso a los recursos del servidor.
